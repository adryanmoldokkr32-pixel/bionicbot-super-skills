# Auto-Documentation & README Architect

This skill scans your project structure and code to create a comprehensive, professional README.md and documentation suite.

## When to Use This Skill

- Initializing a new GitHub repository.
- Updating documentation for an existing project.
- Creating developer guides for internal tools.
- Generating API documentation from source code.

## Instructions

1. List the files and directories in the project.
2. Read core files (e.g., `package.json`, `requirements.txt`, main entry points).
3. Identify the project's purpose, features, and tech stack.
4. Generate a `README.md` with: Title, Description, Features, Installation, Usage, and License.
5. Create a `CONTRIBUTING.md` if needed.

## Examples

- "Generate a README for this Python web scraper project."
- "Document the architecture of my bionicbot-super-skills repo."
