# Cybersecurity & Bug Hunter

This skill proactively audits code repositories for security vulnerabilities, hardcoded secrets, and critical bugs.

## When to Use This Skill

- Pre-deployment security checks.
- Reviewing third-party libraries for vulnerabilities.
- Identifying memory leaks or performance bottlenecks.
- Hardening API endpoints.

## Instructions

1. Scan the codebase for common patterns (e.g., OWASP Top 10).
2. Look for exposed environment variables or hardcoded API keys.
3. Run ad-hoc tests to verify potential vulnerabilities.
4. Document findings with "Severity", "Impact", and "Fix Suggestion".
5. Write automated test cases to prevent regressions.

## Examples

- "Audit my `Ai-buget-sanatate1` repo for any security risks."
- "Search for exposed secrets in my recent commits."
